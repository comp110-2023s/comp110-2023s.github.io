For this exercise, you are going to practice making and instantiating classes in order to simulate a small river ecosystem with bears and fish!

0. Set up your files.

In Visual Studio Code, be sure your workspace is open.

Open the Explorer pane (the icon with two sheets of paper or menu View > Explorer) and expand the Workspace directory.

Right click on the exercises directory and select “New Folder”. Enter the following folder name, being careful to match punctuation:

ex09

Your new folder will contain 4 files.

Right click on ex09 and select “New File”. Enter the following filename, being careful to match punctuation:

fish.py

Right click on ex09 and select “New File”. Enter the following filename, being careful to match punctuation:

bear.py

Right click on ex09 and select “New File”. Enter the following filename, being careful to match punctuation:

river.py

Right click on ex09 and select “New File”. Enter the following filename, being careful to match punctuation:

1. Create Bear Class

class Bear and __init__

For this part, you are going to create and initialize a class to represent the Bear living by the river. In the file bear.py, do the following:

  1. Define a class with the name Bear. It should have the attribute age which is an integer.
  2. Define an __init__ method that only takes self as an argument and initializes the following variable self.age as an int with the value 0

get_age method

Still working in bear.py, define a method inside the Bear class called get_age that takes no parameters other than self and returns self.age.

2. Create Fish Class

class Fish, __init__, and get_age

Now we are going to create a class to represent the fish in the river. In the fish.py file, create a class called Fish. Like Bear, it should have the attribute age which is an integer. Define __init__ and get_age methods inside this class the same way you defined them in the Bear class.

3. Create River Class

Now that you have some animals, you can create and populate a river!

class River

In rivers.py, create a class called River with the following attributes:

  • day: an int that tells you what day of the river’s lifecycle you are modeling
  • bears: a list of Bears that stores the river’s bear population
  • fish: a list of Fish that stores the river’s fish population

__init__ method

Define

Define an init method of River that: - takes as parameters: self, num_fish: int, num_bears: int - initializes self.fish to contain num_fish many Fish - initializes self.bears to contain num_bears many Bears - initializes self.day to be 0

Implement

Implement the __init__ method by constructing opening my_river_sim.py, and constructing a river named my_river with 2 Bears and 10 Fish. (Don’t forget to import the River class!)

view_river() method

Define

Now, you are going to define a method inside your River class called view_riverwhich will print the river status out to you. - It takes no parameters other than self and returns nothing. - It should print the following:

~~~ Day x: ~~~
Fish population: y
Bear population: z

Where x is the current day of the river, y is the number of Fish in the river, and z is the number of Bears in the river.

Implement

At the bottom of my_river_sim.py, try viewing your river by calling my_river.view_river().

When you run python -m exercises.ex09.my_river_sim, the output should be:

~~~ Day 0: ~~~
Fish population: 10
Bear population: 2

4. Simulating “one day” and “one week” in the River

Now that you have a river with Bears and Fish, you can model how their population updates day-to-day.

one_day() methods

Bear and Fish will each have their own one_day method with the following behaviors:

Bear::one_day()

The method one_day() in the Bear class will take no parameters other than self and not return anything. It should increase the value of the age attribute by 1.

Fish::one_day()

The method one_day() in the Fish class will take no parameters other than self and not return anything. It should increase the value of the age attribute by 1.

one_river_day() method

River::one_river_day()

The method one_river_day() in the River class will take no parameters other than self and not return anything. It should behave the following way:

  • Each Bear in bears should implement the one_day() method.
  • If a Bear’s age is > 5 it should be removed from bears.
  • Each Fish in fish should implement the one_day() method.
  • If a Fish’s age is > 3 it should be removed from fish.
  • It should call the print_river() method to display the updated state of the river.
